# Generated Dataset
``extracted_dataset.json`` contains the metrics and data of all the pull requests across the different java projects used in this package.
--- 
---
# Data structure & Description
Each entry is a Json Object representing a Pull Requests with its metadata and diff metrics
#### Primary Fields
| Field | Type | Description                                                             |
| :--- | :--- |:------------------------------------------------------------------------|
| `_id` | `String` | Unique identifier (e.g., `Org_Repo_PR_number`).                       |
| `Meta` | `Object` | Contains GitHub metadata about the project and current pull request        |
| `base` | `Object` | Contains class metrics and method metrics  in the base branch                                               |
| `head` | `Object` | Contains class metrics and method metrics  in the head branch                                                                                                                            |
| `diff` | `Object` | Diff between class metrics and method metrics             |
| `original_comments` | `Array` | List of unsanitized comments of the pull request conversation                                         |
| `cleaned_comments` | `Array` | List of comments from the pull request conversation without noise (markdown quote, code blocks,...)                             |
| `valences` | `Array` | List of sentiment analysis Objects performed on pull request cleaned comments                                         |

---
#### Meta Object content

| Field | Type | Description                                                             |
| :--- | :--- |:------------------------------------------------------------------------|
| `org` | `String` | Name of the repository owner                      |
| `project_name` | `String` | Name of the repository      |
| `PR_id` | `String` | Id of the pull request                                    |
| `PR_number` | `int` | Number of the pull Request                                                                                                                          |
| `PR_url` | `String` | Url to the Pull Request            |
| `Pr_state` | `String` | State of the pull requests (`open` or `closed`)                           |
| `PR_is_merged` | `bool` | Indicates if the pull requests was merged or not                         |
| `PR_additions` | `int` | Number of lines added by the Pull Request
| `PR_deletions` | `int` | Number of lines removed by the Pull Request
| `PR_changed_files` | `int` | Number of files modified by the Pull Request 
| `PR_modified_class_files` | `int` | Number of class files modified by the Pull Request 
| `PR_creation` | `Date` | Date of creation of the Pull Request
| `Base_SHA` | `String` | Sha of the Base branch
| `Head_SHA` | `String` | Sha of the Head branch
| `PR_nb_comments` | `int` | Number of conversation comments in the Pull Request

### Base and Head objects structure
The structure of the Base and Head objects can be seen as the following :
```txt
Base : 
    class1_name
        base_class_metric_1
        base_class_metric_2
        ...
        method1_name
            base_method_metric_1
            base_method_metric_2
            ...
        method2_name
    
    class2_name
    ...
----------------------------

Head : 
    class1_name
        head_class_metric_1
        head_class_metric_2
        ...
        method1_name
            head_method_metric_1
            head_method_metric_2
            ...
        method2_name
        ...

    class2_name
    ...
```
where  `base_class_metric`, `base_method_metric`, `head_class_metric` and `head_method_metric` are obtained from the entries generated by CK analysis (the metrics are the same, refer to [CK README](https://github.com/mauricioaniche/ck) for more details). The classes present in Base and Head are the ones before and after modification by the Pull Request.

### Diff Object Structure
The Diff structure is very similar to Base and Head, except more fields are added : `Was_modified`, `Was_added` and `Was_removed`, 3 booleans indicating if a class and/or method was modififed, added or removed by the Pull Request

3 scenarios possible for classes and methods :

- if the class/method is present in both Base and Head then `Was_modified = true`and `diff_metric` = `head_metric - base_metric`

- if the class/method is present only in Base then `Was_removed = true`and `diff_metric` = `base_metric`

- if the class/method is present only in Head then `Was_added = true`and `diff_metric` = `head_metric`


### Valences Array content

Valences contains Objects giving the positive, negative and neutral value of a Pull Request comment, with the associated compound value returned by the `sentiment_score` method of the ` SentimentIntensityAnalyzer` class from `vaderSentiment` Python library. The `Sentiment` value is given based on compound value (ranging from -1 to 1) :
- `Positive` if `compound` > 0.5
- `Neutral` if `compound` < -0.5
- `Neutral` otherwise
